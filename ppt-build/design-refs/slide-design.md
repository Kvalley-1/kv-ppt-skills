# Jimmy Lin 簡報設計風格指南

> 來源：Jimmy 的「簡報技巧」課程簡報（將捷鬱金香酒店版，約100頁）+ Jimmy 本人修正
> 用途：Claude 製作任何教材前必讀。Jimmy 會持續更新，永遠以最新版為準。

---

## 一、核心哲學與設計直覺

> **「用電影的節奏、極簡的畫面、體驗式的互動，讓觀眾自己得出結論。」**

### 五大原則排序
1. **節奏** > 美觀 — 先決定哪裡停、哪裡快、哪裡炸，再決定放什麼
2. **體驗** > 說明 — 能讓學員自己感受到的，就不要用文字解釋
3. **留白** > 填滿 — 少一個元素，剩下的每個元素都更有力
4. **對比** > 裝飾 — 大小對比、明暗對比、複雜vs簡潔對比，都比加邊框有效
5. **重複** > 新奇 — 同一版型重複使用建立節奏感，不要每頁換一個排版

### 先右腦再左腦
- **先感動，再陳述** — 用情感/畫面引起共鳴，再帶出邏輯/數據
- 簡報是「溝通工具」不是「文件」— 投影片輔助講者，不是取代講者

### 資訊密度
- **每多 1 倍資訊量，多 200% 解讀時間**
- 一張投影片一個概念，一個概念不超過 3 個重點
- 重點字最大 → 次要重點 → 細節文字（字級遞減）

### 視覺優先序
```
影音 > 高清大圖 > Icon符號 > 表 > 文
```

### 控制視線，不給選擇
- 一頁只有一個焦點，讓那個焦點**大到不可能忽略**
- 不是「大標題+小內文」的普通層次，而是**極端字級對比**
- 例：數字「25」佔滿整個畫面 80%，周圍什麼都沒有

### 留白
- 空白不是「沒做完」，是刻意留的呼吸空間
- 頁面上元素越少，剩下的元素越有力
- 不要怕「太空」——怕的應該是「太滿」

## 二、節奏編排（像音樂，不像文件）

### 空與滿的交替
```
空白頁（停頓）→ 大數字衝擊（重拍）→ 互動提問（參與）→ 照片情境（情感）→ 空白頁（呼吸）
```
- 純深色空白頁穿插在章節之間，作為**視覺休止符**
- 連續高密度頁面後一定接一頁「輕」的
- 不是一直講一直講，是「停…停…砰！」

### 漸進式揭露
小測驗系列（數兔子）的節奏設計：
```
提問頁 → 圖片頁（散佈兔子）→ 答案頁（大數字）
提問頁 → 圖片頁（更密集）→ 答案頁
提問頁 → 圖片頁（排成一列，反而更難）→ 答案頁
```
三輪重複同一版型，**訓練學員的參與反射**——舉手動作做三次就變習慣。

## 三、色彩系統

### 雙底色模式（Jimmy 核心偏好）

**Mode A：深色底（主力，70-80% 頁面）**
- 背景：黑色或深色（`#1A1A2E` 黑、`#2D3142` 深灰、或純黑）
- 文字：白色內文 + 金琥珀強調
- 氛圍：專業、沉穩、電影感
- 用途：大多數內容頁、互動頁、大數字衝擊頁、名言金句頁
- 深色底頁負責「感受」

**Mode B：淡米白底（輔助，20-30% 頁面）**
- 背景：淡米白 / 暖白（`#F5F0E8` 近似）
- 文字：深色字體（深灰/黑色）
- 區塊：深色 block 搭配淺色背景，形成層次
- 氛圍：溫暖、柔和、閱讀友善
- 用途：資料呈現、表格內容、需要大量閱讀的頁面
- 米白底頁負責「理解」

**交替節奏**：連續深色頁 5-8 頁後，插入一頁米白底，避免視覺疲勞。

### 主色盤
| 角色 | 色碼（近似值） | 用途 |
|------|---------------|------|
| 深色背景 | `#1A1A2E` ~ `#2D3142`（黑/深灰） | 主力底色 |
| 淡米白背景 | `#F5F0E8`（暖白） | 輔助底色 |
| 主標題/強調 | `#E8A838`（金琥珀/Gold Amber） | 單元標題、重點關鍵字、引言 |
| 深色底內文 | `#FFFFFF`（白色） | 深色背景上的文字 |
| 米白底內文 | `#2D2D2D`（深灰/黑） | 淺色背景上的文字 |
| KValley 品牌綠 | `#C4D600`（Pantone 382C） | 公司 Logo 主色，可用於品牌標識、強調色 |
| KValley 品牌藍 | `#5E8AB4`（Pantone 646C） | 「智谷網絡」文字色，可用於輔助色、圖表 |
| 次要文字 | `#B0B0B0`（淺灰） | 來源標注、輔助說明 |
| 強調輔色 | `#5BA4CF`（天藍） | 數據、進度條、圖表輔色 |

### 配色規則
- **一份投影片不超過 5 個顏色**
- **高對比為王** — 深底配淺字，淺底配深字
- 同色系顯得雅緻 — 避免撞色
- 米白底頁面可用深色 block 做重點區塊，增加層次感

### 金色 = 螢光筆，不是油漆
金色只出現在以下五種位置：
1. 章節標題的「｜」前綴
2. 關鍵數字（統計數據、百分比）
3. 行動指令（「數完的人請舉手！」）
4. 名人引言全文
5. 章節名（｜簡報製作程序 – 美化）

**從不用在一般內文。** 金色到處都是就不特別了。

## 四、照片使用方式（電影分鏡感）

### 不是「配圖」，是「分鏡」
- 同一個人物（Steve Jobs），**不同角度連續 4-5 頁**
- 遠景 → 中景 → 特寫 → 手部特寫
- 每頁只配底部一句字幕式文字
- 把投影片當**分鏡腳本**，不是當文件

### 照片裁切與文字位置
- 人物照：利用人物的**視線方向**引導觀眾讀文字
- 照片與文字的關係是**互相指引**，不是各自獨立

### 全出血照片的用法
- 單元分隔頁：照片佔左 60%，右側深色底 + 一個大關鍵字
- 情境頁：照片滿版，上方深色半透明橫幅放標題
- 絕不把照片縮小放在頁面中間——要嘛滿版，要嘛不放

## 五、數據呈現手法

### 先讓人「感受複雜」，再給「3秒就懂」
1. 第一頁：放原始表格（白底、密密麻麻的數字）→ 讓學員體驗資訊過載
2. 第二頁：用大數字 + 視覺元素重新呈現同一組數據
3. 對比本身就是教學

### 具體轉換手法
| 原始 | 改為 |
|------|------|
| Excel 柱狀圖 | icon 陣列（waffle chart）+ 大數字百分比 |
| 多欄表格 | 3 個大數字 + 進度條 + 一句說明 |
| 密集數據表 | 地圖底圖 + 疊加數據點 |
| 條列式文字 | 大數字置中 + 金色強調 |

### 關鍵數字的呈現
- 數字用金色，放大到佔行寬 30%+
- 單位用白色/灰色，字級縮小
- 說明文字放數字旁邊或下方，不要放在上方

## 六、版型模板（A-J）

### Type A：單元標題頁（Section Divider）
- 左側 ~60% 全出血照片（高清、情境感）
- 右側 ~40% 深色底色
- 右側放一個大關鍵字（白色/金色，超大字）
- 無其他文字裝飾 — 極簡

### Type B：內容頁（標準講授）
- 深色滿版背景
- 左上角：金色標題，前方帶「｜」豎線裝飾
- 標題格式：`｜簡報製作程序 – 美化`
- 內文區：白色文字，大字級（至少 28pt+）
- 底部左側：公司 Logo（KValley + 亞太教育訓練網）

### Type C：大數字衝擊頁（Big Number）
- 深色背景
- 畫面正中央：超大白色數字（佔滿畫面 50%+）
- 無其他元素 — 搭配講師口述

### Type D：互動提問頁（Quiz/Poll）
- 深色背景
- 上方：白色粗體問題
- 下方：金色大字行動指令（字級比問題更大）
- **行動指令永遠是頁面上最大的元素** — 目的是讓人動起來

### Type E：名言金句頁（Quote）
- 深色背景
- 金色大字引言（佔畫面 60%+）
- 右下角：淺灰色署名
- 無其他裝飾

### Type F：全出血照片頁（Full-bleed Photo）
- 照片滿版
- 上方橫幅（深色半透明條）放章節標題（金色）
- 下方疊加白色/金色文字（一句話）

### Type G：Before/After 對比頁
- 深色背景
- 左側「改善前」截圖 + 中間大白色箭頭 → + 右側「改善後」設計
- **不加任何文字解釋「改了什麼」** — 讓視覺差異自己說話

### Type H：流程/步驟頁（Process）
- 深色背景
- 蜿蜒曲線（serpentine）連接步驟
- 每步驟一個圓形 icon + 編號 + 標題
- 圖形用天藍+金色交替

### Type I：數據視覺化頁（Data Viz）
- 避免原始 Excel 圖表直接貼上
- 改用：大數字 + icon 陣列 / 進度條 + 百分比 / 地圖底圖 + 疊加數據
- 關鍵數字用金色，說明用白色

### Type J：課程總覽頁（Agenda/TOC）
- 深色背景，表格式或時間軸
- 金色用於時段區間，白色用於內容描述
- 可搭配 serpentine 流程圖作為反覆出現的導航元素

## 七、排版規則

### 字型
- 中文：思源黑體（Noto Sans TC）或微軟正黑體 — 無襯線
- 英文：Arial / Helvetica — 無襯線
- 標題粗體，內文 Regular

### 字級參考（16:9 投影片）
| 層級 | 字級 | 字重 |
|------|------|------|
| 大數字衝擊 | 200pt+ | Bold |
| 單元標題 | 60-80pt | Bold |
| 頁面標題（｜前綴） | 40-50pt | Bold |
| 內文重點 | 32-40pt | Bold |
| 內文一般 | 24-28pt | Regular |
| 來源/輔助 | 16-18pt | Light |

### 常用分割比例
| 分割 | 用途 |
|------|------|
| 60/40 左右 | 照片+文字、QR Code+說明 |
| 50/50 左右 | Before/After、截圖+說明 |
| 全版單一元素 | 大數字、金句、全出血照片 |
| 上30/下70 | 標題橫幅+主要內容 |

### 標題系統
- 章節標題：金色「｜」前綴 + 粗體，固定在左上角
- 格式：`｜主題名 – 子題名`
- 位置不變，讓學員永遠知道「現在在講什麼」

### Logo
- 底部左下角，小而低調，不搶內容的視覺權重

## 八、故事線三法 + 簡報製作六步驟

### 製作六步驟
```
01 分析 — 聽眾、環境、目的
02 發想 — 蒐集素材、激盪點子
03 歸納 — 篩選、去蕪存菁
04 製作 — 故事線、組織成簡報
05 美化 — 視覺設計、多媒體
06 預演 — 練習、練習、再練習
```

### 故事線三法
| 方法 | 注意力曲線 | 適合 |
|------|-----------|------|
| 問題-方案法 | U 型 | 銷售提案、問題導向培訓 |
| 時間軸法 | 階梯式上升 | 專案報告、歷程回顧 |
| 功能特色法 | 鋸齒形 | 產品發表、功能介紹 |

## 九、互動設計技巧

- **小測驗系列**：漸進難度 part 1/2/3 — 訓練參與反射
- **舉手投票**：大字問題 + 金色行動指令
- **倒數計時**：大數字頁搭配口述
- **Steve Jobs 式口述**：全螢幕人物照 + 底部一句金句（連續分鏡）
- **Before/After**：不解釋，讓視覺差異自己說話
- **破冰**：Use video, 問互動性問題, 把道具帶過來, Tell a story, 現場投票
- **能量管理**：鼓勵大家動起來, Share #, Add music, 使用適當的動畫, 視覺化數據

## 十、設計檢查清單

製作投影片時逐項確認：
- [ ] 背景色使用雙模式：深色（黑/深灰）為主，淡米白為輔
- [ ] 顏色不超過 5 色
- [ ] 每頁只有 1 個概念
- [ ] 重點字級明顯大於其他文字（極端對比）
- [ ] 有全出血照片作為單元分隔
- [ ] 數據用視覺化呈現（非表格截圖）
- [ ] 互動頁行動指令是頁面最大元素
- [ ] Logo 在底部左下角，小而低調
- [ ] 標題帶「｜」豎線前綴
- [ ] 先情感再邏輯的順序
- [ ] 空白頁穿插作為節奏休止符
- [ ] 金色只用在五種位置（標題/數字/指令/引言/章節名）
- [ ] 照片要嘛滿版要嘛不放，不縮小放中間
