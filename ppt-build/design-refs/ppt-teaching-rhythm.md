# 教學用 PPT 節奏設計原則（Day 1 200 張實戰萃取）

> **建立**：2026-04-26 — 從台水 Day 1 PPT 成品（200 張，~/Downloads/台水Day1_完整版_v2.pptx）逆向萃取。
> **目的**：未來幫 Jimmy 生成教學用 PPT 時的節奏設計參考，避免規格頁直接 1:1 渲染（會單調）。
> **配套**：[apple-ppt-design-system.md](apple-ppt-design-system.md)（視覺規範） + [ppt-templates/](ppt-templates/)（模板庫） + [scripts/pptx_apple_style.py](../scripts/pptx_apple_style.py)（helper library）
> **核心觀念**：**1 個教案規格節點 ≈ 3-5 張實際投影片**，差距由 cue card / section title / build slides / 過場 / 視覺鋪陳補滿。

---

## 〇、為什麼要這份原則？

**踩雷紀錄（2026-04-26）**：盤點 Day 2 PPT 時直接從教案規格抓 63 個節點 → Jimmy 指出太少，因為**沒考慮 cue card 跟章節標題**。

**事實**：Day 1 教案規格 62 頁，但實際 PPTX **200 張**（擴張 3.2 倍）。差距全部在 cue card、section title、build slides、漸進披露、互動觸發。

**原則**：教學用 PPT 不是「文件版投影片」，是「**現場節奏的視覺載體**」。每張只服務一個瞬間，講師按一下就帶出下一個動作。

---

## 一、9 種投影片類型（從 Day 1 200 張歸納）

### A. Section Title（章節大字過場）— INK 底
**形狀數**：4-7
**特徵**：`─ SECTION 0X ─` + 副標 + 主題大字 + 時間戳
**用 helper**：`section_title(slide, num, small_title, main, time_str, page)`
**用量**：每段開頭 1 張，全天約 10-15 張（Day 1 約 12 張）

```
Slide 027：09 / ─  SECTION 02  ─ / 學習心理學 / 為什麼學不起來？
Slide 061：13 / ─  SECTION 03  ─ / 值得傳的是什麼 / 知識・資訊・心法 / 10:05-10:20 · 15 分鐘
Slide 137：41 / ─  SECTION 10  ─ / 8 向度 Canvas 訪談 / 隱性知識萃取 / 13:25-14:20 · 55 分鐘
```

### B. Cue Card（指令 / 提示卡）— 白底或 INK 底
**形狀數**：1-3（極少元素）
**特徵**：1 個短句或 1 個動作指令 + 大字置中（30-60pt）
**用量**：佔 Day 1 約 20%（~40 張）— 這是 Day 1 最被忽略的「節奏黏著劑」

**典型 cue card 範例**：
```
動作指令：    「開始拋」「分享」「先挑夥伴」「先產生訪談大綱」
時間提示：    「2 分鐘討論」「3 分鐘討論」「準備好錄音 app」
反思觸發：    「為何需要暖場？」「優點有什麼？」「可改善點？」
過場提問：    「人家都寫好了，要萃取什麼？」「具體要做什麼加工？」
情緒喚起：    「換大家試試看」「再多一點點什麼，可以更好」
轉場銜接：    「形成 / 下一步 / 加工決策」「有方向了，然後呢」
任務交付：    「記得上傳文件到 NotebookLM」「交換角色，再一次」
```

**設計鐵律**：
- ❌ 不要塞背景說明 — cue card 越乾淨越好
- ✅ 一張一個動作，講師按一下就推進
- ✅ 字級 30-60pt（後排看得到）
- ✅ 與前後內容頁形成節奏差異

### C. Build Slides（漸進披露）— 同一頁拆成 2-5 張
**特徵**：同一個視覺基底，每張多加 1 個元素（文字、圖、bullet）
**用量**：佔 Day 1 約 15%（~30 張）

**典型 build 模式**：
```
Pattern 1: 「拋問題 → 收答 → 揭曉」
  062 DIKW 表 → 063 a/b/c/d 選項 → 064「沒有結論/重點」（揭曉）
  066 選項 → 067「沒有結論/重點」
  068 選項 → 069「照著做就好」
  070 選項 → 071「照著做就好」

Pattern 2: 「每輪一張」（多輪活動）
  005 第一關規則 → 006 ROUND 01 → 007「開始拋」 → 008 結果
  009 ROUND 02 → 010「2 分鐘討論」 → 011 規則 → 012「開始拋」→ 013 結果
  014「3 分鐘討論」→ 015 ROUND 03 → 016「開始拋」→ 017 結果

Pattern 3: 「鋪陳 → 揭曉」
  029「猜歌名」歌詞 → 030 多一段 → 031「閱讀五線譜」（提示）→ 032 圖

Pattern 4: 「視覺逐步建構」
  043 標題 → 044「+E Y M O R M」→ 045 空白過場 → 046「+ M E M O R Y」
```

**設計鐵律**：
- 用「重複頁面 + 漸進加元素」，不是「一頁塞滿」
- 每張只多一個元素（文字 / 圖 / bullet 都行）
- 每張對應講師的一個動作（拋、收、揭、補）

### D. 內容主頁（白底完整資訊）
**形狀數**：15-40
**特徵**：頁標 44pt + 副標 16pt + hairline + 多卡片組合
**用 helper**：`title_header()` + 自訂卡片
**用量**：佔 Day 1 約 25%（~50 張）

**規格頁 → 主頁對應**：
- 教案規格中標明「內容講授」、「對照表」、「框架圖」的，渲染成 1 張內容主頁
- 同一規格頁如果內容多，可拆成 2-3 張 build slides

### E. Hero / Insight / Wrap（金句衝擊頁）— INK 底
**用 helper**：`hero_dark(slide, label, lines_spec, page, footer)`
**用量**：每段結尾 1 張，全天 8-12 張

**典型 Hero 範例**：
```
Slide 019  ─  DEBRIEF  ─
           我們花了十幾分鐘學會，
           下一個人怎麼學的快？
           方法教得了，手感教不了——
           除非你知道怎麼把手感變成看得見的東西。

Slide 060  ─  INSIGHT  ─
           學習不是直線——
           每個「會了」背後，
           都有幾百次失敗的嘗試。

Slide 126  ─  MORNING WRAP  ─
           知識不會自己走路。
           需要有人把它挖出來。
           那個人，就是你。

Slide 199  ─  DAY 1 WRAP  ─
           今天你做到了：
           ✓ 體驗了知識傳承的難
           ✓ 分辨了知識的三個層次
           ...
```

**Label 詞庫**（依時段選用）：
- 段落結尾：`DEBRIEF` / `INSIGHT` / `RECAP` / `KEY POINT`
- 上午結尾：`MORNING WRAP` / `MORNING CLOSE`
- 下午中段：`AFTERNOON BRIDGE`
- 全天結尾：`DAY 1 WRAP` / `DAY 1 CLOSE`
- 行動啟動：`ACTION` / `LET'S GO`

### F. Action 啟動頁（亮藍底）
**用 helper**：`hero_light(slide, label, lines_spec, page, footer)`
**用量**：每個活動段啟動時 1 張，全天 5-8 張

```
Slide 149  ─  ACTION  ─
           開始訪談！
           Canvas 在手  ·  Gemini 備好  ·  錄音打開
```

### G. Prompt 大卡頁
**用 helper**：`prompt_card(slide, title, prompt_text, use_case, output, accent, page)`
**用量**：每個 Prompt 1 張，當有 N 個 prompts 就 N 張（Day 1 是 5 個進階 prompt = 5 張）

### H. Visual-heavy（純圖頁）
**形狀數**：1-3（其中 1 張大圖）
**特徵**：純圖 + 短說明（10 字內）
**用量**：佔 Day 1 約 12.5%（~25 張）

**用途**：
- 認知負荷實驗（記憶字串、看圖）
- 互動猜歌（歌詞圖）
- 案例展示（截圖）
- 進階 Prompt 結果（AI 產出截圖）
- 過場視覺（單一畫面 + 標題）

### I. 雙卡對比頁（× vs ✓）
**用 helper**：`dual_compare(slide, title, subtitle, bad_label, ..., good_label, ...)`
**用量**：依教案需求，全天 3-5 張

```
Slide 158  為什麼不能「自由訪談」？
           × 不推薦                    ✓ 推薦
           自由訪談                     Canvas 訪談
           ・話題飄走，30 分鐘後忘了    ・8 欄引導框架，結構化每次訪談
           ・前輩答「你做久就會了」     ・主題不跑偏，受訪者被問對問題
           ・挖不到判斷邏輯             ・搭配 AI 追問，補深度
```

## 二、規格頁 → 實際投影片的擴張公式

### 平均擴張率：3.2 倍（從 Day 1 200 / 62 算出）

### 各類型擴張規律

| 規格頁類型 | 擴張倍數 | 拆成什麼 |
|---|:---:|---|
| **章節分隔頁** | 1x | 1 張 section title（不擴張）|
| **內容講授頁** | 1.5-2x | 1 張內容主頁 + 0-1 張 build slide |
| **互動拋問頁** | 3-5x | 拋問題 + 收答選項 + 揭曉 + 補充 |
| **多輪活動段** | 5-10x | 每輪：規則 + 啟動 + 計時 + 結果 |
| **段落結尾** | +1x | 多 1 張 hero_dark 金句頁 |
| **活動啟動** | +1x | 多 1 張 hero_light Action 頁 |
| **轉場 / 過場** | 散見 | cue card 1-3 張 |

### 計算公式

```
實際投影片數 ≈ 規格節點 × 1.5  (純內容部分)
            + 互動段數 × 4    (互動 + 揭曉)
            + 段落數 × 2      (section title + hero wrap)
            + 活動段數 × 1    (action 啟動頁)
            + cue card 數     (現場節奏黏著劑)
```

## 三、節奏編排原則（光桌層級）

### 鐵律：連續兩張不能同色同版型

**Day 1 實際節奏範例**（Slide 001-020）：
```
001 封面（白）
002 全天 Timeline（白，特殊版型）
003 章節大字（白底，特殊版型）        ← 變化
004 規則說明（白底內容頁）
005 規則細節（白底內容頁）
006 ROUND 01 提示（白底，build 1）    ← 變化
007 「開始拋」（白底 cue card）        ← 變化
008 結果頁（白底 build 2）
009 ROUND 02 提示（白底 build 1）
010 「2 分鐘討論」（cue card）         ← 變化
...
019 DEBRIEF 金句（INK 底）            ← 強烈節奏切換
```

**節奏切換信號**：
- 連續 3-4 張同類型 → 必須換版型
- 段落結束 → 必接 INK 底 hero（不論前面是什麼）
- 互動結束 → 必接金句或下段 section title
- 休息前 → 段落結尾金句已過，直接休息頁

### 上午 vs 下午節奏差異

**上午** (09:00-12:00)：
- 節奏密集，互動多（馬戲團、知識三層案例）
- cue card 比例高（推進現場動作）
- Build slides 多（漸進披露）

**下午** (13:00-16:00)：
- 節奏較緩，內容主頁多（知識萃取、Prompt 5 招）
- Prompt 大卡集中
- 內容主頁佔比上升

## 四、實作流程（給未來 Claude 用）

### 步驟 1：拿到教案規格 → 盤點節點

讀完 Day X 教案，把每個「節點」列出來（含 RD 編號、時段、規格描述）。

### 步驟 2：分類擴張

對每個節點判斷：

| 節點類型 | 擴張方式 |
|---|---|
| 章節開頭（如 RD-25 開場） | + 1 張 section title |
| 純內容講授（如 RD-31 PPT-A1） | × 1-2（看內容厚度） |
| 互動拋問（如 KC 拆解金字塔示範） | × 3（拋 + 答 + 揭） |
| 多輪活動（如 Prompt 卡牌 R1/R2/R3） | × 4-5（每輪：規則 + 動作 + 結果） |
| 教學示範段（如 Zillow 回扣） | × 5（鋪陳 + 設問 + 播 + 暫停 + 收答） |
| 段落結尾 | + 1 張 hero_dark 金句 |
| 活動啟動 | + 1 張 hero_light Action |

### 步驟 3：補 cue card

每個段落（不論長短）都會用到 cue card。常見位置：
- 「開始 [動作]」啟動指令
- 「[X 分鐘] 討論」時間提示
- 「[單一提問]」反思觸發
- 「先 [準備動作]」前置準備
- 「換 [角色] 試試看」轉場
- 「然後呢？」「具體要做什麼？」過場提問

### 步驟 4：節奏檢查

逐張檢查：
- ☐ 連續兩張不同色／不同版型？
- ☐ 每段結尾有 hero_dark 金句？
- ☐ 每個活動啟動有 hero_light Action？
- ☐ 章節過場都用 INK 底？

### 步驟 5：總張數估算

基準：教案節點數 × 3.2 = 預期投影片數

如果差超過 ±20%，重新檢視 cue card 跟 build slides 的拆分粒度。
